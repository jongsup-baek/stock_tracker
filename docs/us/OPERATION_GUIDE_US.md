## 2. Daily Report 생성 프로세스

### 2.1 사용자 요청
```
"US IRA 1월 30일 Daily Report 만들어줘"
```

### 2.2 Claude가 데이터 읽기
- GitHub `data/us/stock_*.json` - 당일 주가 데이터 (**전체 종목**)
- GitHub `docs/us/PORTFOLIO_US.md` - 관심 종목
- GitHub `docs/us/OPERATION_MANUAL_US.md` - 매매 규칙
- Notion 매매 히스토리 - 매수가, 수량, 거래 내역

### 2.3 분석 및 생성
- 정배열 조건 확인 (전체 종목)
- 매수 조건 4가지 점검 (전체 종목)
- 보유 종목 평가손익 계산
- 익절/손절 기준 확인

### 2.4 Notion Daily Report DB에 저장
- Name: "US IRA 2026년 1월 30일"
- 날짜 속성 자동 설정
- 넘버링 시스템으로 구조화된 내용

## 4. Daily Report 템플릿

```

(전체 종목 테이블: 종가, 전일비, MA5, MA10, MA20)

## 2. 매수 조건 점검
### 2.1 매수 조건 분석
(전체 종목: ①정배열 ②종가<MA5 ③MA10±2% ④거래량↓10%)

(4/4 충족 종목 또는 주의 필요 종목)

## 3. 포트폴리오 현황
### 3.1 자금 현황
### 3.2 보유 현황
### 3.3 익절/손절 기준

## 4. Action Items
### 4.1 내일 장 시작 전
### 4.2 모니터링 가격

## 5. 자금 관리 점검

## 6. 특이사항

## 7. 더블 체크
```

## 5. 더블 체크 프로세스

### 5.1 데이터 완전성 체크

| 체크 항목 | 확인 |
|----------|------|
| 전체 종목 데이터 확인 | ✅ |
| 매수 조건 4가지 모두 점검 | ✅ |
| 보유 종목 익절/손절 기준 계산 | ✅ |
| 자금 관리 (매수 가능 여부) 확인 | ✅ |
| 전일 대비 변동률 계산 | ✅ |

### 5.2 계산 정확성 체크

| 체크 항목 | 공식 |
|----------|------|
| 정배열 | MA5 > MA10 > MA20 |
| 종가 < MA5 | 종가가 MA5보다 낮은지 |
| MA10 ±2% 범위 | MA10 × 0.98 ~ MA10 × 1.02 |
| 거래량 감소 | 당일 거래량 < 10일 고점 × 0.9 |
| 익절가 | 매수가 × 1.15 |
| 손절가 | 매수가 × 0.93 |
