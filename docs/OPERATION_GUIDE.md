## 2. Daily Report 생성 프로세스

### 2.1 사용자 요청
Claude.ai Cowork에게 Daily Report 요청:
```
"1월 27일 Daily Report 만들어줘"
```

### 2.2 Claude가 데이터 읽기
- GitHub `data/stock_*.json` - 당일 주가 데이터 (**6개 종목 전체**)
- GitHub `docs/PORTFOLIO.md` - 관심 종목
- GitHub `docs/OPERATION_MANUAL.md` - 매매 규칙 (v1.4)
- Notion 매매 히스토리 - 매수가, 수량, 거래 내역

### 2.3 분석 및 생성
- 정배열 조건 확인 (6개 종목 전체)
- 매수 조건 4가지 점검 (6개 종목 전체)
- 보유 종목 평가손익 계산 (거래세 0.5% 반영)
- 익절/손절 기준 확인

### 2.4 Notion Daily Report DB에 저장
- Name: "2026년 1월 27일" (날짜만, 요일 없음)
- 날짜 속성 자동 설정
- 넘버링 시스템으로 구조화된 내용

## 3. 데이터 저장 위치

| 항목 | 저장 위치 | 역할 | 업데이트 주기 |
| --- | --- | --- | --- |
| **주가 데이터** | GitHub `/data/` | 원본 데이터 (JSON) | 매일 자동 (GitHub Actions) |
| **관심 종목** | GitHub `/docs/PORTFOLIO.md` | 관심 종목 목록 (Public OK) | 수동 (필요 시) |
| **운영 문서** | GitHub `/docs/` | 상세 규칙 (AI용) | 수동 (필요 시) |
| **매매 히스토리** | Notion 페이지 | 매수/매도 기록 (Private) | 거래 체결 시 |
| **Daily Report** | Notion DB | 일일 분석 결과 | 매일 수동 (Claude 요청) |
| **투자 규칙 요약** | Notion 메인 페이지 | 간단 참조 (사람용) | 수동 (규칙 변경 시) |

> 💡 **데이터 분리**: 공개 가능한 정보(주가, 규칙)는 GitHub에, 민감한 정보(매수가, 수량)는 Notion에 저장합니다.

> 🔒 **민감 정보 보호**: 매수가, 수량 등의 민감한 정보는 Notion에만 저장하고, GitHub는 Public으로 공개해도 안전합니다.

## 4. Daily Report 템플릿

### 4.1 필수 섹션 (넘버링 시스템)

```
## 1. 시장 요약
### 1.1 전체 종목 현황
(6개 종목 전체 테이블: 종가, 전일비, MA5, MA10, MA20)

## 2. 매수 조건 점검
### 2.1 매수 조건 분석
(6개 종목 전체: ①정배열 ②종가<MA5 ③MA10±2% ④거래량↓10%)
### 2.2 ~ 2.X 개별 종목 상세
(4/4 충족 종목 또는 주의 필요 종목)

## 3. 포트폴리오 현황
### 3.1 자금 현황
(총 투자금, 현금, 투자금, 보유 종목)
### 3.2 보유 현황
(매수가, 수량, 투자금, 현재가, 단순 수익률, 실질 평가손익)
### 3.3 익절/손절 기준
(매수가, 익절가 +15%, 손절가 -7%, 현재 상태)

## 4. Action Items
### 4.1 내일 장 시작 전
(구체적 수치 포함: 종목, 가격, 수량, 금액)
### 4.2 모니터링 가격
(익절 목표, 손절 기준)

## 5. 자금 관리 점검
(매수 시 필요 금액, 잔여 현금 계산)

## 6. 특이사항
(당일 주요 이벤트, 신호 변화)

## 7. 더블 체크
(아래 체크리스트 참조)
```

### 4.2 매수 조건 테이블 형식

| 종목 | ①정배열 | ②종가<MA5 | ③MA10±2% | ④거래량↓10% | 결과 |
|------|---------|-----------|----------|-------------|------|
| 삼성전자 | ✅/❌ | ✅/❌ (수치) | ✅/❌ (%) | ✅/❌ (%) | X/4 |

**4/4 충족 시**: 🔥 매수 신호!

### 4.3 보유 현황 테이블 형식

| 종목 | 매수가 | 수량 | 투자금 | 현재가 | 단순 수익률 | 실질 평가손익 |
|------|--------|------|--------|--------|------------|--------------|
| | | | | | | **(거래세 0.5% 반영)** |

## 5. 더블 체크 프로세스

Daily Report 작성 완료 후, 반드시 아래 항목을 확인합니다.

### 5.1 데이터 완전성 체크

| 체크 항목 | 확인 |
|----------|------|
| 6개 종목 전체 데이터 확인 | ✅ |
| 매수 조건 4가지 모두 점검 | ✅ |
| 보유 종목 익절/손절 기준 계산 | ✅ |
| 자금 관리 (매수 가능 여부) 확인 | ✅ |
| 전일 대비 변동률 계산 | ✅ |

### 5.2 계산 정확성 체크

| 체크 항목 | 공식 |
|----------|------|
| 정배열 | MA5 > MA10 > MA20 |
| 종가 < MA5 | 종가가 MA5보다 낮은지 |
| MA10 ±2% 범위 | MA10 × 0.98 ~ MA10 × 1.02 |
| 거래량 감소 | 당일 거래량 < 10일 고점 × 0.9 |
| 익절가 | 매수가 × 1.15 |
| 손절가 | 매수가 × 0.93 |
| 실질 평가손익 | 단순 수익률 - 0.5% (거래세) |

### 5.3 Action Items 구체성 체크

- [ ] 매수 검토 시: 종목, 예상가격, 수량, 금액 명시
- [ ] 손절 주의 시: 현재 손실률, 손절가까지 여유 명시
- [ ] 익절 접근 시: 현재 수익률, 익절가까지 거리 명시

### 5.4 누락 방지 체크

- [ ] 어제 미체결 주문 후속 처리 확인
- [ ] 정배열 깨진 종목 별도 표시
- [ ] 손절 기준 접근 종목 경고

---

> ⚠️ **중요**: 이 더블 체크 프로세스는 Daily Report 마지막 섹션 "7. 더블 체크"에 결과를 기록합니다.
